This version accepts the list of attributes as a dictionary, key-value pairs. The key is attribute names, and value can be of any type supported by JSON (string, int, bool, etc...).

Authorizations:
JSON_Web_Token
path Parameters
id
required
string <uuid>

Account Id

header Parameters
X-Sym-OrganizationId
required
string <uuid>
Example: 51840c3a-4bb9-4b4a-b8d9-0ca1f93a76a7

Organization UUID of the client making the request

Request Body schema:

Body contains list of properties which are key-value pairs. The keys are attribute IDs as seen in the data-target-defintions endpoint.

Array of objects (CustomerAttribute)
object (AccountFormMetaData)

Responses

localhost

http://api.local.symendinc.com/customer/v1/accounts/{id}/actions/update-immediate

development

https://api-dev.symend.io/customer/v1/accounts/{id}/actions/update-immediate

test

https://api-test.symend.io/customer/v1/accounts/{id}/actions/update-immediate
